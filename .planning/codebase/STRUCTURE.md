# Codebase Structure

**Analysis Date:** 2026-01-25

## Directory Layout

```
[project-root]/
├── .auto-claude/        # Internal tools/artifacts for Claude development
├── .husky/              # Git hooks
├── apps/                # Contains independent applications
│   ├── ai-service/      # Node.js/TypeScript backend service (AI/crawling)
│   ├── dashboard/       # Next.js web application (frontend)
│   └── ios/             # Swift iOS application (mobile)
├── comfyui/             # Configuration/workflows for ComfyUI (AI art/workflow)
├── convex/              # Convex.dev backend-as-a-service configurations/code
├── docs/                # Documentation
├── n8n/                 # Workflow automation tool configurations
├── packages/            # Reusable libraries and shared code
│   ├── constants/       # Shared constants
│   ├── convex/          # Convex-specific utilities/integrations
│   ├── crawler-types/   # Types definitions for crawlers
│   ├── logger/          # Logging utilities
│   ├── test-utils/      # Utilities for testing
│   ├── types/           # General shared type definitions
│   └── utils/           # General utility functions
├── .planning/           # Internal planning documents (generated by agents)
└── scripts/             # General project scripts
```

## Directory Purposes

**apps/:**

- Purpose: Holds all independent, deployable applications. Each subdirectory is a distinct application.
- Contains: Application source code, configuration, and build artifacts.
- Key files: `apps/*/package.json`, `apps/*/src/`, `apps/*/tsconfig.json`

**apps/ai-service/:**

- Purpose: Backend service responsible for AI processing, data crawling, and API exposure.
- Contains: TypeScript source code, crawler implementations, API routes.
- Key files: `/apps/ai-service/src/index.ts`, `/apps/ai-service/src/lib/crawlers/`

**apps/dashboard/:**

- Purpose: Web-based user interface for interacting with the system.
- Contains: React/Next.js components, pages, styles, hooks.
- Key files: `/apps/dashboard/src/app/page.tsx`, `/apps/dashboard/src/components/`

**apps/ios/:**

- Purpose: Native iOS mobile application.
- Contains: Swift source code, Xcode project files, assets.
- Key files: `/apps/ios/Pathfinding/Pathfinding.xcodeproj/`, `/apps/ios/Pathfinding/Pathfinding/Core/`

**packages/:**

- Purpose: Monorepo for shared code, utilities, and type definitions used across multiple applications.
- Contains: TypeScript/JavaScript libraries, helper functions, interface definitions.
- Key files: `/packages/*/src/index.ts`, `/packages/*/package.json`

**convex/:**

- Purpose: Contains code and configurations related to the Convex backend-as-a-service.
- Contains: Convex functions, schema definitions.
- Key files: `convex/_generated/` (generated code), `convex/*.ts` (Convex functions)

**.planning/:**

- Purpose: Stores planning and analysis documents generated by automated agents.
- Contains: Markdown files detailing codebase analysis, plans, and concerns.
- Key files: `.planning/codebase/ARCHITECTURE.md`, `.planning/codebase/STRUCTURE.md`

## Key File Locations

**Entry Points:**

- `/apps/ai-service/src/`: Primary entry for the AI backend service.
- `/apps/dashboard/src/app/page.tsx`: Main entry component for the Next.js dashboard.
- `/apps/ios/Pathfinding/Pathfinding/PathfindingApp.swift` (or similar): Main entry for the iOS application.

**Configuration:**

- `/package.json`: Root monorepo dependencies and scripts.
- `/apps/*/package.json`: Application-specific dependencies and scripts.
- `/packages/*/package.json`: Package-specific dependencies and metadata.
- `/tsconfig.json`, `/apps/*/tsconfig.json`, `/packages/*/tsconfig.json`: TypeScript configurations.
- `convex/*.ts`: Convex function and schema definitions.

**Core Logic:**

- `/apps/ai-service/src/lib/`: Core logic for the AI service.
- `/apps/dashboard/src/`: Core UI components and pages for the dashboard.
- `/apps/ios/Pathfinding/Pathfinding/Core/`: Core logic and networking for the iOS app.
- `/packages/*/src/`: Source code for shared libraries.

**Testing:**

- `/apps/*/src/**/*.test.ts`: Test files co-located with source in AI service/dashboard.
- `/packages/*/src/**/*.test.ts`: Test files for shared packages.
- `/packages/test-utils/`: Shared utilities for testing.

## Naming Conventions

**Files:**

- `kebab-case` for non-component files (e.g., `my-utility.ts`).
- `PascalCase.tsx` for React components (e.g., `MyComponent.tsx`).
- `camelCase.ts` or `kebab-case.ts` for TypeScript/JavaScript source files.
- `PascalCase.swift` for Swift files.

**Directories:**

- `kebab-case` for most directories (e.g., `ai-service`, `test-utils`).
- `PascalCase` for Swift modules/groups (e.g., `Core`, `Network`).

## Where to Add New Code

**New Feature (Web Dashboard):**

- Primary code: `/apps/dashboard/src/app/` (for pages), `/apps/dashboard/src/components/` (for reusable components).
- Tests: Co-located `*.test.tsx` files.

**New API Endpoint (AI Service):**

- Implementation: `/apps/ai-service/src/routes/` or new service in `/apps/ai-service/src/services/`.
- Tests: Co-located `*.test.ts` files.

**New iOS Feature:**

- Implementation: `/apps/ios/Pathfinding/Pathfinding/Features/` (if a `Features` directory exists, otherwise within relevant `Core` or new module).
- Tests: Co-located `*.test.swift` files within the iOS test target.

**New Shared Utility:**

- Implementation: `/packages/utils/src/` for general utilities, or a new package under `/packages/` if it's a significant, self-contained module.
- Tests: `/packages/utils/src/**/*.test.ts` or `packages/<new-package>/src/**/*.test.ts`.

## Special Directories

**.auto-claude/:**

- Purpose: Contains internal artifacts, insights, specs, and memory used by the Claude agent for project management and development.
- Generated: Yes.
- Committed: Yes (for tracking agent activity).

**.planning/:**

- Purpose: Stores planning and analysis documents generated by automated agents.
- Generated: Yes.
- Committed: Yes.

`convex/_generated/`:

- Purpose: Contains client code and type definitions automatically generated by Convex based on the defined backend functions and schema.
- Generated: Yes.
- Committed: Yes (typically, as it's needed for client-side type checking and API calls).
