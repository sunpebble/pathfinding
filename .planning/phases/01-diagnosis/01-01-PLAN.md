  <action>
Create capture.ts with:

1. `DiagnosticCapture` interface:
   - platform, url, timestamp
   - snapshot (full accessibility tree)
   - networkRequests (array of NetworkRequest)
   - consoleMessages (string array)
   - navigationTime, contentLoadTime (timing data)
   - parseResult (CrawlResult | null), parseErrors (string array)

2. `waitForContentStable(maxWait = 10000, stabilityWindow = 500)`:
   - Take snapshots every stabilityWindow ms
   - Return true when two consecutive snapshots have same length and length > 1000
   - Return false if maxWait exceeded

3. `captureForDiagnosis(platform: string, url: string)`:
   - Navigate to URL with 30s timeout
   - Call waitForContentStable()
   - Take verbose snapshot
   - List network requests (xhr, fetch, document)
   - Return DiagnosticCapture object (parseResult/parseErrors left null for later)

Import from mcp-client.ts: navigateTo, takeSnapshot, listNetworkRequests, sleep
Export: DiagnosticCapture, captureForDiagnosis, waitForContentStable

Create report.ts with:

1. `FailureCategory` type union:
   - acquisition:blocked, acquisition:login_required, acquisition:timeout, acquisition:captcha, acquisition:empty
   - parsing:no_content, parsing:no_images, parsing:partial, parsing:selector_miss
   - success

2. `detectAntiBotIndicators(content: string)`:
   - Check for captcha patterns: /验证|captcha|verify/i
   - Check for login wall: /登录|sign.?in|login/i but NOT /退出|logout/i
   - Check for blocked: /blocked|forbidden|access denied/i
   - Check for empty: content.length < 500
   - Return Record<string, boolean>

3. `categorizeFailure(capture: DiagnosticCapture)`:
   - First check acquisition issues (empty, captcha, login_required, blocked)
   - Then check parsing issues (no_content, no_images, partial)
   - Return 'success' if all expected data present

4. `generateDiagnosticReport(capture: DiagnosticCapture)`:
   - Call categorizeFailure
   - Format human-readable report with:
     - Platform, URL, timestamp
     - Timing data (nav, content load)
     - Snapshot length and preview (first 500 chars)
     - Anti-bot indicators
     - Network request count
     - Failure category and recommended action

Create index.ts that re-exports from capture.ts and report.ts.
